# DevOps Project 1 – Deploy a Web App with Docker

## Objective

In this project you will practice the basic DevOps workflow using:

- **Git**
- **Linux**
- **Docker**

You will clone an existing project, containerize the application, run it, and document your deployment steps.

---

## Repository

Clone the project repository below:
https://github.com/pkicloudconsulting/project1.git


---

## Project Requirements

You are required to:

1. Clone the repository
2. Create your own **feature branch**
3. Make sure your branch is **up to date with master**
4. Write a **Dockerfile**
5. Build a **Docker image**
6. Run a **Docker container**
7. Verify the web application in your browser
8. Push your changes to your **origin feature branch**
9. Create a **Pull Request (PR)** with documentation

---

## Expected Outcome

By the end of this project you should be able to:

- Use **Git for version control**
- Work with **feature branches**
- Create a **Dockerfile**
- Build and run **Docker containers**
- Deploy a **web application locally**
- Document your deployment steps in a **Pull Request**

---

## Submission Instructions

After successfully deploying the web application:

1. Take a **screenshot of the web app running in your browser**
2. Post the screenshot in the **class WhatsApp group**
3. Push your changes to your **feature branch**
4. Create a **Pull Request to the master branch**

---

## Pull Request Description

Your Pull Request must include:

- The **steps you took to deploy the application**
- The **commands you used**
- Any **issues encountered and how you resolved them**
